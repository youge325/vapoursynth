卷积
===========

.. function:: Convolution(vnode clip, float[] matrix[, float bias=0.0, float divisor=0.0, int[] planes=[0, 1, 2], bint saturate=True, string mode="s"])
   :module: std

   执行空间卷积。

   以下是 3x3 的工作原理。3x3 邻域中的每个像素乘以 *matrix* 中相应的系数。九次乘法的结果相加，然后除以 *divisor*。接下来加上 *bias*，结果舍五入到最接近的增大整数则。如果此整数结果为负 *saturate* 参数为 False，乘以 -1。最后，结果被钳制到格式的有效值范围内。

   *夹子*
      要处理的片段。必须具有 8 到 16 位之间的整数采样类型，或 32 位的浮点采样类型。如果存在其他格式的帧，将返回错误。

   *矩阵*
      卷积系数。
      
      当 *mode* 为 "s" 时，必须是 9 或 25 个数字的内存，分别对应 3x3 或 5x5 。

      当 *mode* 不为 "s" 时，必须是 3 到 25 个数字的负载，元素数为奇数。

      系数值必须在 -1023 和 1023 之间（含）。当输入为整数格式时，系数将四舍五入为整数。

      以下是 *matrix* 元素与 3x3 邻域中像素的关系对应：

         1 2 3
         4 5 6
         7 8 9

      其他类型的视觉也遵循相同的原理。*matrix* 的中间元素始终对应中心像素。

   *偏见*
      添加到卷积最终结果的值（在将结果钳制到格式的有效值范围之前）。

   *除数*
      将公式输出除初步值（在添加 *bias* 之前）。

      如果此参数为 0.0（默认值），舒适的输出将除以 *matrix* 元素之和，如果和为 0 则除以 1.0。

   *飞机*
      指定要处理的平面。未处理的平面将被简单复制。

   *饱和*
      最终结果会被钳位到格式的有效值范围 (0 .. (2**bit深度)-1)。因此，如果此参数为 True，负值会等于 0。如果此参数为 False，则对绝对值进行钳位并返回。

   *模式*
      选择格式类型。可能的格式为 "s" 表示方形，"h" 表示水平，"v" 表示垂直，"hv" 或 "vh" 表示水平和垂直组合。

   如何应用接着于 Avisynth 的 Blur(1) 的简单模糊：
   
   .. code-block:: python
   
      Convolution(matrix=[1, 2, 1, 2, 4, 2, 1, 2, 1])
   
   如何应用直接于 Avisynth 的 Blur(1.58) 的更强的模糊：

   .. code-block:: python
      
      Convolution(matrix=[1, 1, 1, 1, 1, 1, 1, 1, 1])
