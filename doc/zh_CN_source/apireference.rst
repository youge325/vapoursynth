常见陷阱
###############

有一些与线程和设计相关的小陷阱需要注意。其中大多数通常不是问题，但这里有一个有时需要注意的小清单。

通用API
-----------
不能将一个核心拥有的对象（片段、函数等）作为参数传递给另一个核心中的滤镜。需要手动进行完整的深拷贝。这通常不是问题，因为每个滤镜图不应需要多个核心。

插件
-------
插件代码的多线程程度可能比表面上看起来更高。*VapourSynthPluginInit* 是唯一保证不会运行的函数。这意味着过滤器的构造函数和解析构造函数可能会为多个实例运行。如果需要初始化共享数据，请使用适当的同步机制。

*GetFrame* 函数结构复杂一些，请参见常量参考。但请注意零件性是按实例计算的。即使过滤器是 *fmUnordered* 或 *fmSerial*，其他实例也可能同时进入 *GetFrame*。

关于应使用哪种模式存在两个常见的误区。一个简单的规则是永远不宜使用*fmSerial*。需要锁定的源过滤器（在*arInitial*时返回的过滤器）应使用*fmUnordered*。

保留帧属性
#########################

所有帧都包含键值对映射。建议使用 a-z、A-Z、0-9 以驼峰命名法命名这些属性。有一类特殊的以 _ 开头的键，具有下面严格定义的含义。如果不确定，可以不设置这些键。将它们设置为下面未指定的值是致命错误。

int _ChromaLocation（色度位置）

   YUV格式中的色度采样位置。
   
   0=左，1=中，2=左上，3=上，4=左下，5=下。
   
int _Range（色彩范围）

   全范围或有限范围（PC/TV范围）。主要用于YUV格式。

   1=全范围，0=有限范围。
   
int _ColorRange（旧范围字段）

   已废弃，请使用 API R4.2 中引入的 *_Range*。这些属性为了坚固兼容而保持链接。

   全范围或有限范围（PC/TV范围）。主要用于YUV格式。

   0=全范围，1=有限范围。
 
int _Primaries（色域原色）

   ITU-T H.273 表 2 中规定的色域原色。
   
int _Matrix（矩阵系数）

   ITU-T H.273 表 4 中规定的矩阵系数。
   
int _Transfer（传输特性）

   ITU-T H.273 表 3 中规定的传输特性。
   
int _FieldBased（场结构标记）

   帧是否由两个独立的场组成（隔行扫描）。

   0=帧（逐行），1=底场优先，2=顶场优先。

浮动_绝对时间

   源滤镜报告的帧绝对时间戳（以秒为单位）。应仅由源滤镜设置且不应修改。所有依赖帧长度的操作应使用持续时间。

int _DurationNum, int _DurationDen（帧时长分子/分母）

   帧的持续时间，以有理数表示（单位为秒）。修改帧率的滤镜也应更改这些值。

   此分数应始终被约分。
   
宾特_梳子

   帧是否需要后处理，通常由场匹配滤镜提示。

int _Field（字段来源）

   如果帧是由 core.std.SeparateFields 等函数生成的，此属性指示使用了哪个场来生成此帧。

   0=来自底场，1=来自顶场。

字符串_PictType

   描述帧类型的单个字符。使用常见的IPB字母，但其他字母也可用于具有附加帧类型的格式。

int _SARNum, int _SARDen（采样宽高比分子/分母）

   像素（采样）宽高比，以有理数表示。

bint _SceneChangeNext（下一帧是否场景切换）

   如果为 1，此帧是当前场景的最后一帧。下一帧开始新场景。

bint _SceneChangePrev（当前帧是否场景起始）

   如果为 1，此帧开始一个新场景。

框架_Alpha

   可以使用此属性将分段的 alpha 通道逐帧附加到分段上。
